# Датасеты проекта spatialaug

Собрано 7 датасетов разных типов и масштабов. Цель — разнородное покрытие, которое позволит проверить, что библиотека для интерполяции и аугментации геоданных работает не только на одном типе задачи. Все датасеты — открытые либо публичные.

---

## 1. ФНС геочеки (основной)

**Источник:** платформа поставки данных ФНС России, https://geochecki-vpd.nalog.gov.ru/geochecks (публичный API без авторизации)

**Объем:** 3 531 hex-агрегат по 11 городам, 952 KB

**Что в данных:** агрегированная информация о работе контрольно-кассовой техники, разбитая по сетке гексагонов H3 (Uber). Каждый hex содержит около 30 признаков:

- Количество ККТ (контрольно-кассовых аппаратов) и референсное значение
- Средний / усеченный / медианный чек в рублях, плюс референс
- Доля наличных платежей и доля наличных чеков
- Порядковая интенсивность чеков и выручки (1–10)
- Топ-3 категорий товаров (одежда, кафе, медицинские и т.п.)
- 24-элементный вектор активности по часам суток (процент активности на каждый час)
- Флаги: торговый центр / редкий объект / интернет-торговля
- Период агрегации (одна неделя)

**Роли в проекте:**

- **Основной датасет** для главных экспериментов библиотеки. Прямое соответствие банковской и финтех-постановке проекта: размещение банкоматов, оценка ритейл-зон, скоринг районов.
- **Мультицелевая пространственная интерполяция:** на одном датасете 7+ непрерывных целевых переменных.
- **Реальный механизм приватных пропусков (MNAR):** ФНС официально зануляет некоторые поля (`avg_bill`, `median_bill`, `top_categories`) на hex-ах с малым числом ККТ, чтобы скрыть данные отдельных мерчантов. Это регуляторный механизм приватных пропусков.
- **Встроенный сигнал доменного сдвига:** колонки `*_ref` дают референсное значение (видимо, по административной единице — Москва не равна Московской области).
- **Ареальная структура с иерархией:** H3 дает бесплатную систему соседства (6 соседей у каждой соты) и иерархию (родитель / потомок).

**Покрытие — 11 городов разной плотности:**

| Категория | Города | Hex-ов |
|---|---|---|
| плотные | Москва, Санкт-Петербург | 1710 |
| средние | Казань, Екатеринбург, Новосибирск, Челябинск | 939 |
| разреженные | Воронеж, Краснодар, Калининград | 746 |
| редкие | Магадан, Якутск | 136 |

**Лицензия:** публичные открытые данные ФНС.

**Период:** 2026-04-13 — 2026-04-19 (одна неделя).

## 2. Российская недвижимость 2018–2021 (вторичный основной)

**Источник:** HuggingFace, datasets/daniilakk/russian_real_estate

**Файл:** `data/real_estate/labeled_final.parquet`

**Объем:** 2 283 078 строк, 76 MB

**Что в данных:** объявления о продаже жилой недвижимости с сайтов агрегаторов:

- Точные географические координаты (`geo_lat`, `geo_lon`)
- Цена в рублях
- Регион РФ (84 уникальных)
- Характеристики квартиры: площадь, кухня, этаж, этажность дома, число комнат, тип здания
- Дата объявления

**Что добавлено пайплайном:**

- `label` — ценовой уровень относительно региональной медианы: budget (49.8%), mid_range (39.7%), premium (9.4%), luxury (1.0%)
- `confidence` — уверенность разметки

**Роли в проекте:**

- **Вторичный основной** для экспериментов на точечных данных (в отличие от ФНС, где данные ареальные). Библиотека должна работать с обоими типами.
- **Сравнение по регионам:** Москва и СПб против разреженных регионов. Доменный сдвиг в ценах: Москва — медиана 9.9 млн руб., среднестатистический регион — 2.5 млн руб.
- **Многорегиональная статистика:** 84 региона позволяют исследовать перенос вариограмм кригинга между разнотипными зонами.

**Лицензия:** CC.

## 3. OSM-инфраструктура 5 городов (ковариаты)

**Источник:** OpenStreetMap через Overpass API

**Файл:** `data/osm/osm_russia_cities.parquet`

**Объем:** 20 266 строк, 612 KB

**Что в данных:** координаты точек интереса (POI) из OSM:

- Банки: 5 300+
- Банкоматы: 7 643+
- Магазины: 7 300+

**Города:** Москва, Санкт-Петербург, Новосибирск, Екатеринбург, Казань.

**Роли в проекте:**

- **Ковариаты для ко-кригинга:** добавляем плотность банкоматов и магазинов как ковариату к модели интерполяции цены недвижимости или ритейл-активности.
- **Валидация для ФНС:** есть прямая совместимость по городам — можно проверить корреляцию плотности OSM-POI с `kkt_count` ФНС.

**Лицензия:** ODbL-1.0.

## 4. BDMO — муниципальная статистика Росстата (ареальный)

**Источник:** tochno.st / Росстат, https://tochno.st/datasets/bdmo

**Файл:** `data/bdmo/bdmo_services.parquet`

**Объем:** 1 502 378 записей, 4 MB

**Что в данных:** база данных муниципальных образований РФ с 2005 года. Сейчас собрана секция 1 — платные услуги населению, разбитые по ОКТМО (общероссийский классификатор территорий муниципальных образований).

**Роли в проекте:**

- **Ареальные / полигональные данные** — единственный датасет, где значения привязаны к административным единицам, а не к точкам.
- **Долгосрочное временное покрытие:** 2005 — настоящее, что позволяет в дальнейшем добавить временное расширение.

**Лицензия:** CC BY 4.0.

## 5. ДТП — дорожно-транспортные происшествия (кросс-доменный)

**Источник:** tochno.st / ГИБДД, https://tochno.st/datasets/traffic_accidents

**Файл:** `data/traffic/traffic_accidents.parquet`

**Объем:** 1 616 059 записей, 50 MB

**Что в данных:** все зарегистрированные ДТП в РФ за 2015–2026 с точными координатами и атрибутами:

- `latitude`, `longitude`
- Дата и время
- Тип ДТП, степень тяжести
- Условия (погода, освещение, дорожные условия)

**Роли в проекте:**

- **Кросс-доменная валидация для библиотеки.** Совершенно другой домен (безопасность, не финтех) с теми же пространственными методами.
- **Альтернативный пример MNAR:** в малых населенных пунктах фиксация ДТП хуже, и пропуски в данных коррелируют с плотностью населения.

**Лицензия:** CC BY 4.0.

## 6. RFSD — российская финансовая отчетность (кросс-доменный)

**Источник:** HuggingFace, irlspbru/RFSD

**Файл:** `data/rfsd/rfsd_sample_50k.parquet`

**Объем:** 50 000 компаний (выборка из 60M полного датасета), 2.1 MB

**Что в данных:** бухгалтерская отчетность российских юридических лиц. 213 финансовых показателей на компанию плюс координаты местонахождения.

**Роли в проекте:**

- **Кросс-доменная валидация номер 2:** еще один не финтех-в-узком-смысле домен — экономика юридических лиц с пространственной привязкой.
- **Высокая размерность признаков (213):** удобно для ML-методов с гео-признаками.

**Лицензия:** CC BY 4.0.

## 7. Sberbank Russian Housing Market (многопризнаковый, Москва)

**Источник:** Kaggle, muhammadhanzlatahir/sberbank-russian-housing-market

**Файл:** `data/sberbank/train.parquet`

**Объем:** 30 471 сделка, 15 MB

**Что в данных:** сделки купли-продажи недвижимости в Москве с очень богатой инфраструктурной разметкой — 292 признака на объект:

- Привязка к району Москвы (`sub_area`)
- Цена сделки
- Расстояния до объектов инфраструктуры (метро, школы, парки, торговые центры)
- Демография и социальный профиль района
- Экологические показатели

**Роли в проекте:**

- **Многопризнаковое пространственное исследование:** 292 признака на одну точку — высокая размерность гео-признаков.
- **Ареальные данные на районном уровне:** sub_area — это район Москвы, что дает еще один пример ареальных данных, но мельче чем BDMO (район входит в муниципалитет).

**Лицензия:** проверена для академического использования.
